## 2024-03-12 - Critical SQL Injection in DuckDB Analytics Queries
**Vulnerability:** SQL Injection via f-strings in DuckDB parameter queries. User inputs (start_time, end_time, and node_ids) were directly concatenated into DuckDB SQL queries without parameterization, allowing arbitrary DuckDB function execution.
**Learning:** Even internal analytical functions using DuckDB to query Parquet files require parameterized queries if any inputs trace back to an API or user. To parameterize IN clauses with an unknown number of items in DuckDB with Python, it's necessary to dynamically generate placeholders (`?, ?, ?`) that match the length of the list, and then extend the execution parameters list.
**Prevention:** Always use parameterized queries `conn.execute(query, params)` with dynamically generated placeholders (`placeholders = ",".join(["?"] * len(nodes))`) for IN clauses, rather than joining values directly into the SQL string. Ensure timestamps and external parameters are correctly CAST via `CAST(? AS TIMESTAMP)`.
